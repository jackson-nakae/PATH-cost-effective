# PATH Cost Effective

This repository contains the current PATH report workflow used to generate the interactive HTML report, the static PDF report, and the optimization/data-preparation code that will be integrated into WEPPcloud as the new module **PATH cost effective**.

The module is intended to expose the same inputs used by the report and optimizer:

- landscape selection
- hillslope sediment yield threshold
- outlet sediment discharge threshold
- slope / burn severity filters
- treatment names, costs, quantities, and fixed costs
- paths to hillslope, contrast, and spatial input data

## What this repository produces

- A fully parameterized optimization run
- An interactive HTML report with maps, sliders, tables, and downloadable threshold data
- A PDF report with the same scenario results
- Generated CSV / PKL threshold sweep outputs for reuse on later renders

## Current workflow

The reports are built from the same parameter set and should be rendered together.

1. The user or WEPPcloud module supplies thresholds, filters, treatment costs, and file locations.
2. `PATH_data_prep.py` loads and standardizes hillslope, contrast, and spatial data.
3. `PATH_CE.py` runs the cost-effective optimization.
4. `PATH_plot.py` computes the threshold sweep and plotting helpers.
5. `PATH_CE_Report_Universal.qmd` builds the HTML report and caches threshold sweep data.
6. `PATH_CE_Report_PDF.qmd` builds the PDF report from the same inputs and cached data.
7. `render_both_reports.py` renders HTML first, then PDF, so both outputs stay synchronized.

## Quick start

1. Create and activate a Python environment.
2. Install dependencies:

```bash
pip install -r requirements.txt
```

3. Render both reports together:

```bash
python render_both_reports.py --landscape pacificcreek --sdyd-threshold 10 --sddc-threshold 35000
```

If `--landscape` is omitted, the script auto-detects the landscape from the files in the workspace.

## Report entry points

### HTML report

`PATH_CE_Report_Universal.qmd`

- interactive HTML report
- interactive map and slider-based threshold exploration in section 3
- threshold sweep results are written to `static/downloads/`
- intended for browser viewing and collaboration

### PDF report

`PATH_CE_Report_PDF.qmd`

- static report version for archival and distribution
- uses the same optimization inputs and threshold analysis results as the HTML report
- currently built from the same landscape / threshold parameter set

### Unified render helper

`render_both_reports.py`

- renders HTML first
- then renders PDF
- passes the same landscape and thresholds to both reports
- also exports runtime environment variables so the Quarto notebooks resolve the same values consistently

## Core Python files

- `PATH_CE.py`: optimization logic, including `ce_select_sites_flexible()`
- `PATH_data_prep.py`: loads and standardizes hillslope / contrast / spatial inputs
- `PATH_plot.py`: threshold sweep and plotting helpers used by both reports

## Handoff / packaging workflow

`create_handoff_bundle.py` packages the report and assets for collaborator handoff.

Use this when preparing a shareable bundle for WEPPcloud integration or offline review.

Example:

```bash
python create_handoff_bundle.py --landscape pacificcreek --sdyd-threshold 10 --sddc-threshold 35000
```

Outputs are written under `dist/`.

## WEPPcloud integration notes

The upcoming WEPPcloud module, **PATH cost effective**, should provide the inputs required by the reports and optimizer:

- threshold values
- landscape name
- treatment names and costs
- optional filter values
- paths or payloads for hillslope, contrast, and spatial data

The module should call the same workflow used here so that:

- the optimization runs once
- the HTML and PDF reports stay synchronized
- threshold sweep results can be reused instead of recomputed when available

## Generated outputs

The HTML report can create cached threshold sweep artifacts in `static/downloads/`, including:

- `{landscape}_threshold_analysis_results_generated.csv`
- `{landscape}_threshold_analysis_results_generated.pkl`

These files are used to keep later renders and the interactive slider map consistent with the originally computed threshold sweep.

## Notes for collaborators

- The HTML report is the primary interactive deliverable.
- The PDF report should be rendered from the same landscape and thresholds.
- The reports expect the same data schema that `PATH_data_prep.py` produces.
- The interactive map in section 3 depends on the threshold sweep results generated by the report run.
